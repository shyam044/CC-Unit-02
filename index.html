1) Implementation levels of Virtualization

Virtualization separates hardware from software to improve efficiency and flexibility

i) ISA Level

  • ISA stands for the Instruction Set Architecture
  • ISA Level Virtualization means using one ISA to run another ISA
  • It is Achieved through
    • Code Interpretation
    • Binary Translation

ii) Hardware Level

  • It happens directly on the physical hardware like CPU, memory, I/O Devices
  • It creates virtual versions of physical hardware
    • Virtual CPU
    • Virtual Memory
    • Virtual I/O Devices

iii) OS Level

  • OS stands for Operating System
  • All Containers share the same OS kernel which makes them lightweight
  • Example:
    • Docker
    • LXC[Linux Container]

iv) Library Level

  • Virtualization happens at the library or API level
  • It controls the communication between application and system through API hooks
  • Example:
    • Wine: Windows applications run on Linux

v) Application Level

  • Individual Applications are virtualized not the whole system
  • It is also called as Process Level Virtualization
  • Example
    • JVM[Java Virtual Machine] to run the Java Programs
    • .NET CLR to run the .NET Applications

2) Full Virtualization, Para Virtualization, Hardware Virtualization, Software Virtualization

Full Virtualization:

  • It doesn't know it is running in the virtual Machine.
  • The Guest OS is not modified.
  • OS Runs Normally
  • VMM(also called as Hypervisor) catches the privileged instructions.
  • Full Virtualization is slower
  • Full Virtualization is more complex for hypervisor
  • Full Virtualization is less secure

Para Virtualization:

  • It knows it is running in the virtual Machine.
  • The Guest OS is modified.
  • OS uses hypercalls.
  • It uses hypercalls to directly communicate with the Hypervisor.
  • Para Virtualization is faster.
  • Para Virtualization is less complex for hypervisor.
  • Para Virtualization is more secure

Hardware Virtualization:

  • Virtualization is supported by Hardware.
  • The Guest OS is not required to be modified.
  • Hardware handles Virtualization tasks.
  • Hardware Virtualization shows higher performance.
  • Hardware Virtualization is less complex for hypervisor.
  • Hardware Virtualization is more secure.

Software Virtualization:

  • Virtualization is supported by Software.
  • The Guest OS is not required to be modified.
  • Software handles Virtualization tasks.
  • Software Virtualization shows lower performance.
  • Software Virtualization is more complex for hypervisor
  • Software Virtualization is less secure.

3) Types of Virtualization:

i) Application Virtualization:

  • It allows you to use the software without installing its application on your computer.
  • They run on a remote server.
  • You can connect it through Internet.
  • All data are stored on servers not on your computer.

ii) Network Virtualization:

  • It creates multiple virtual networks by using just one physical network.
  • It creates virtual versions of Routers, Switches, Firewalls, etc.. by using software
  • A company can create different virtual networks for different departments.

iii) Desktop Virtualization:

  • It allows logging into your personal computer from anywhere.
  • They run on a remote server.
  • You can connect it through Internet.
  • Easy to update the software for all users at once.

iv) Storage Virtualization:

  • It combines multiple storage systems into a single storage system.
  • Continues to work smoothly even if one storage device fails.
  • Storage Virtualization helps in backup, recovery, etc...

v) Server Virtualization:

  • It divides one physical server into multiple virtual servers.
  • It creates and manages the virtual servers by using a special software called as Hypervisor.
  • Server Virtualization also helps in backup, recovery, etc...

vi) Data Virtualization:

  • It gathers the data from different devices into one place.
  • It creates a unified view of the data
  • You can access the data through a single Interface.

5) Containerization:

  • Containerization is a lightweight form of Virtualization that runs the applications in an isolated container.
  • Containerization inserts the virtualization layer inside the OS Kernel.
  • Containerization shares the same OS Kernel.
  • Applications running in one container do not affect applications running in other containers.

    • Containerization tools:

    Some popular Containerization tools are:

    • Docker: Docker is used for creating and deploying the containers.
    • LXC: LXC stands for Linux Container, LXC is for OS level virtualization.

7) Resource Scheduling, Resource Allocation, Resource Management Tools in Virtual Clusters

Resource Scheduling:

  • The VMM (also called as hypervisor) is responsible for sharing the resources to different virtual machines.
  • The Hypervisor uses a Scheduling Algorithm to know how the resources are allocated.
  • Types of Scheduling Algorithm:
    • Proportional share:
      • It is a Fair Share.
      • Each VM gets resources based on its assigned weights.
    • Priority-Based Scheduling:
      • VM gets resources based on Priority levels.
      • Higher priority VMs get resources before lower priority ones.

Resource Allocation:

  • Resource allocation means assigning physical hardware (like CPU, memory, storage) to virtual machines.
  • Key Concepts:
    • Dynamic Scaling:
      • It automatically adds or removes resources based on the workloads.
    • Load Balancing:
      • It distributes the workloads across multiple servers.
      • Load Balancing Algorithm is mainly of two types:
        • Static Load Balancing Algorithm
        • Dynamic Load Balancing Algorithm

Resource Management Tools:

8) Serverless Computing in the Context of Virtualization

  • Serverless computing is an evolution of virtualization where developers can run the application without maintaining the servers.
  • In Serverless Computing, developers only write and deploy the code.
  • Serverless Computing is built by using these Virtual Technologies:
    • Virtual Machines (VMs)
    • Containers
    • Virtual Clusters

Benefits of Serverless Computing:

i) No Server Management:

  • Developers can run the application without maintaining the servers.

ii) Developer Efficiency:

  • Developers only write and deploy the code.

iii) Runs on Demand:

  • Code only runs when a particular event is triggered.

iv) Cost Efficient:

  • You only pay for the time your code runs.

Examples:

  • AWS Lambda
  • Google Cloud Functions
  • Microsoft Azure Functions
